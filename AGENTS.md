## 📋 Назначение репозитория

**Sortify** - это модуль для Magisk и KernelSU на Android, который автоматически сортирует загруженные файлы по категоризированным папкам. Модуль работает в фоновом режиме и периодически сканирует папку загрузок, перемещая файлы в соответствующие категории на основе их расширений.

## 📂 Структура файлов и компонентов

### 1. **module.prop** - Манифест модуля
- Определяет ID, имя, версию и описание модуля
- Используется Magisk/KernelSU для идентификации и отображения модуля
- **Ключевые поля:**
  - `id=sortify` - уникальный идентификатор
  - `version=4.3.0` - отображаемая версия
  - `versionCode=14` - числовой код версии для обновлений
  - `author=xCaptaiN09` - автор модуля

### 2. **config.json** - Конфигурация
Конфигурационный файл для настройки поведения модуля:
```json
{
    "enabled": false,                 // 🆕 Включение/выключение модуля (по умолчанию выключен)
    "interval": 1000,                 // Интервал сканирования в секундах
    "base_path": "/sdcard/Sortify",   // Куда сортировать файлы
    "download_path": "/sdcard/Download", // Откуда брать файлы
    "recursive": true,                // Рекурсивное сканирование
    "max_depth": 5,                   // Максимальная глубина сканирования
    "exclude_folders": ["WhatsApp", "Music", "Screenshots"] // Исключаемые папки
}
```

**🆕 Изменения в форке:**
- Параметр `enabled` позволяет включать/выключать модуль без перезагрузки
- Дефолтный `interval` увеличен до 1000 секунд для экономии батареи
- Дефолтные `exclude_folders` обновлены под типичные use cases

### 3. **service.sh** - Фоновый сервис
- Запускается автоматически при загрузке Android
- Ожидает готовности хранилища (`/sdcard`)
- Работает в бесконечном цикле с настраиваемым интервалом
- Парсит `config.json` и экспортирует переменные для `action.sh`
- Ведет лог (`sortify.log`) с автоматической ротацией (последние 200 строк)
- Копирует лог в `base_path` для удобного доступа

**Ключевые функции:**
- `get_json_value()` - парсинг строковых и числовых значений из JSON
- `get_json_bool()` - парсинг булевых значений
- `wait_until_storage()` - ожидание монтирования хранилища

### 4. **action.sh** - Логика сортировки
Основной скрипт, выполняющий сортировку файлов. Версия **4.3.0** включает:

**Новые возможности v4.3.0:**
- Автоматическое определение модулей Magisk через проверку наличия `module.prop` в ZIP-архивах
- Специальная папка `Magisk/` для модулей

**Возможности v4.3:**
- Рекурсивный поиск с контролем глубины (`recursive`, `max_depth`)
- Автоматическое исключение `base_path` если он внутри `download_path`
- Пользовательские исключения через массив `exclude_folders`

**Категории файлов:**
```bash
Documents/ : pdf, doc, docx, txt, xls, xlsx, ppt, pptx, csv
Images/    : jpg, jpeg, png, gif, bmp, webp, heic, heif, svg
Videos/    : mp4, mkv, avi, mov, webm, flv, mpeg, mpg, 3gp
Audio/     : mp3, m4a, flac, wav, ogg, opus, aac, wma
Archives/  : zip, rar, 7z, tar, gz, bz2, iso (кроме модулей Magisk)
Apps/      : apk, xapk, apks
Magisk/    : zip-файлы с module.prop внутри (новое в v4.3.0)
Others/    : все остальные файлы
Duplicates/: файлы с конфликтующими именами
```

**Исключаемые файлы:**
- Скрытые файлы (начинающиеся с `.`)
- Временные файлы (`.crdownload`, `.partial`, `.tmp`)
- Файлы в исключенных папках

**Ключевые функции:**
- `get_json_value()` - парсинг значений из config.json
- `get_json_bool()` - парсинг булевых значений
- `get_json_array()` - парсинг массивов из config.json
- `is_magisk_module()` - проверка ZIP-файла на наличие module.prop
- `log_msg()` - логирование с временной меткой
- `move_files()` - перемещение файлов с обработкой дубликатов

**Алгоритм работы:**
1. Чтение конфигурации из `config.json`
2. Построение списка исключений (авто + пользовательские)
3. Создание структуры папок в `base_path`
4. Последовательная сортировка по категориям:
   - Документы → Images → Videos → Audio
   - ZIP-файлы (проверка на модули Magisk) → Magisk/
   - Архивы → Apps
   - Остальные файлы → Others
5. Обработка дубликатов (перемещение в `Duplicates/`)

### 5. **customize.sh** - Установщик
- Выполняется при установке модуля через Magisk Manager
- Создает структуру папок в `base_path`
- Устанавливает права доступа на скрипты (0755)
- Показывает информацию о конфигурации и новых функциях

### 6. **uninstall.sh** - Деинсталлятор
- Выполняется при удалении модуля
- Останавливает фоновые процессы (`pkill -f sortify`)
- По умолчанию НЕ удаляет отсортированные файлы (закомментировано)
- Логирует удаление в `sortify.log`

### 7. **webroot/index.html** - 🆕 Native WebUI
**Новая функция форка MeteorBurn!**

Single-page веб-интерфейс для полного управления модулем через KsuWebUI или WebUI X.

**Основные секции:**
- **Module Status** - переключатель включения/выключения с предупреждением
- **General Settings** - настройка интервала, путей, рекурсии, глубины
- **Folder Exclusions** - управление исключаемыми папками (multiline textarea)
- **Sortify Log** - 🆕 Live просмотр логов с кнопками Refresh/Clear
- **File Categories** - информационная секция о категориях
- **Debug Info** - скрытая панель для отладки (display: none)

**Технические детали:**
- **Config Loading:** Использует `cat | tr '\n' ' '` для чтения многострочного JSON
- **Config Saving:** Heredoc с `cat > file << 'EOFCONFIG'` для безопасной записи
- **Log Loading:** `while read loop` для чтения всех строк лога
- **ksu.exec API:** Все операции через JavaScript API KernelSU WebUI
- **Responsive Design:** Адаптивный дизайн с dark theme
- **Auto-scroll:** Логи автоматически прокручиваются к концу
- **Scrollbar:** min-height: 400px, max-height: 600px, overflow-y: scroll

**Преимущества:**
- Не требует отдельного веб-сервера
- Работает полностью в KSU WebUI/WebUI X
- Мгновенные изменения без перезагрузки (для `enabled` параметра)
- Удобный просмотр логов без ADB

## 🔧 Разработка и модификация

### Тестирование изменений
1. Модифицируйте необходимые скрипты (`action.sh`, `service.sh`)
2. Создайте ZIP-архив модуля с правильной структурой
3. Установите через Magisk Manager
4. Проверьте логи в `/data/adb/modules/sortify/sortify.log`

### Добавление новых категорий файлов
1. Откройте `action.sh`
2. Добавьте новое расширение в соответствующую переменную (например, `DOC_EXT`)
3. Или создайте новую категорию:
   ```bash
   NEW_EXT="ext1 ext2 ext3"
   mkdir -p "$BASE_PATH/NewCategory"
   move_files "$BASE_PATH/NewCategory" $NEW_EXT
   ```

### Изменение логики сортировки
- Основная логика находится в функции `move_files()` в `action.sh`
- Используйте команду `find` с фильтрами для поиска файлов
- Обрабатывайте исключения через проверку `EXCLUDE_LIST`
- Логируйте важные действия через `log_msg()`

### Совместимость с Android busybox
Скрипты используют только базовые команды POSIX shell:
- `find`, `grep`, `sed`, `cut`, `tr`, `mv`, `mkdir`
- Избегайте bash-специфичных конструкций (массивы, `[[`, `+=`)
- Используйте простые циклы `while read` вместо `for file in $(find ...)`
